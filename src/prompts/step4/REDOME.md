# ISRステップ4（成果物・HTML記事）人間検品用リフレクションガイドライン

## 1. 振り返りの目的
ステップ4で生成されたHTML記事は、ユーザー（読者）に届く**最終成果物**です。
人間がこのガイドラインに沿って記事を振り返ることで、「まとめサイトのような退屈な正論記事」になっていないか、動画の魂が宿った「体験型Web記事（ISR）」として完成しているかを1〜2分で評価します。

---

## 2. 4大チェックポイント（検品基準）

### ① トーンの憑依度と「余白」の確認（マイルド＆ナラティブ）
投稿者の思考を100%トレースし、読者に寄り添う語り口になっているかを確認します。

*   **[ OK ]** 「〜という傾向にあります」「〜なところが多いです」「〜なのかもしれません」といった、読者に考えさせる柔らかい表現（非断定表現）が地の文のベースになっている。
*   **[ NG ]** 「〇〇です」「〇〇ですね」といった強い断定表現が乱発され、上から目線の「お説教」や「解説」に聞こえる。

### ② ことわざの「ステルス融合」チェック
抽象的な心理分析を分かりやすくするための「ことわざ・格言」が、自然に溶け込んでいるかを確認します。

*   **[ OK ]** ストーリーや解説の流れ（地の文）の中に、ことわざが違和感なく組み込まれている。
*   **[ NG ]** `ことわざ：因果応報` のように機械的なラベルが付いている、あるいは箇条書きで独立して紹介されていて、流れがブチ切れている。

### ③ エモーショナル・クォートの「聖域化」チェック
ステップ2で抽出した「本人の生の叫び（キラーフレーズ）」が正しく扱われているかを確認します。

*   **[ OK ]** 指定のHTMLタグ（`<blockquote class="emotional-quote">`）で囲まれており、文字起こしから1文字も変更されずにそのまま配置されている。前後に「なぜ本人がここで熱くなったのか」の心理的解説がある。
*   **[ NG ]** AIが気を利かせて、キラーフレーズを綺麗な標準語や要約文に書き換えてしまっている。

### ④ 体験型ギミック（図解・ミニゲーム）の連動性
読者が飽きずに読み進め、自身のバイアスに気づく仕組みが機能しているかを確認します。

*   **[ OK ]** 
    - `本質の構造化` でMermaidコードと「いらすとや」の的確な構図指示がある。
    - `体験型ミニゲーム` のシチュエーションがリアルで、AIのフィードバックが読者の「安易な正義感」を優しく戒める内容になっている。
*   **[ NG ]** ミニゲームの選択肢が極端すぎてゲームになっていない。またはフィードバックがただの正論の押し付けになっている。

## 3. 人間が下すべき最終ジャッジ（アクション）

成果物の状態に応じて、運用者は以下の3つのいずれかのアクションを選択します。

| 成果物の状態 | 判定 | 次のアクション |
| :--- | :--- | :--- |
| 断定表現がなく、ことわざがストーリーに溶け込んでいる。ミニゲームも面白い。ノイズの復活もない。 | **【 パブリッシュ（GO） 】** | HTMLをそのままCMS（WordPress等）に流し込み、**記事を公開**する。 |
| 全体的なクオリティは高いが、ことわざの繋がりが1箇所だけ不自然だったり、HTMLタグの閉じ忘れがある。 | **【 クイック修正（TWEAK） 】** | 人間の手でその部分だけを**数秒で手動リライト・修正して公開**する。 |
| まとめサイト的な客観トーン（三人称）に戻ってしまっている。または、消したはずの本の宣伝が復活している。 | **【 差し戻し（REJECT） 】** | ステップ3のクンジングデータを確認するか、ステップ4のプロンプトの「絶対遵守ルール」を強調して**再生成**する。 |

---

## 運用のコツ：ノイズの「ゾンビ化」に注意
AIは非常に親切なため、ステップ3でせっかく消した「本の宣伝」や「格ゲーのコラボ告知」を、ステップ4の執筆時に**「読者のために親切で付け足しておこう！」と勝手に捏造して復活（ゾンビ化）させる**ことが稀にあります。

記事の末尾（「明日からの1ステップ」の周辺など）に、動画独自の普遍的な考察とは関係のない「事務連絡」が紛れ込んでいないかだけ、人間の目で最後にサッと見張ってあげてください。そこさえクリアしていれば、極上のISR記事の完成です。
